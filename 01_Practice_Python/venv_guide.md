# 파이썬 가상환경 .venv 란?

파이썬 개발 시 `.venv`는 가상환경(Virtual Environment)을 구성할 때 생성하는 일반적인 폴더명이다.
가상환경은 프로젝트별로 독립된 파이썬 실행 환경과 패키지를 격리하여 관리하기 위해 사용한다.

## 주요 개념 정리

### 가상환경의 필요성 (패키지 버전 충돌 방지)
하나의 컴퓨터(Global 환경)에 모든 패키지를 설치하면, 프로젝트마다 요구하는 패키지의 버전이 다를 경우 충돌이 발생할 수 있다.
예를 들어 프로젝트 A는 특정 라이브러리의 2.0 버전을, 프로젝트 B는 4.0 버전을 필요로 할 수 있다.
각 프로젝트마다 개별적인 `.venv`를 생성하면 환경이 완전히 격리되므로 이러한 의존성 문제(Dependency Hell)를 효과적으로 방지할 수 있다.

## 가상환경 사용 방법

### 1. 가상환경 생성
프로젝트 디렉토리 내에서 다음 명령어를 통해 가상환경을 생성한다.
```bash
python -m venv .venv
```

### 2. 가상환경 활성화
운영체제에 따라 활성화 명령어가 다르다.
- Mac / Linux:
```bash
source .venv/bin/activate
```
- Windows:
```bash
.venv\Scripts\activate
```
활성화가 완료되면 터미널 프롬프트 앞에 `(.venv)`가 표시된다. 이후부터 설치하는 모든 패키지는 이 환경 안에만 저장된다.

### 3. 가상환경 접속 확인 (경로 확인)
가상환경이 올바르게 적용되었는지 파악하기 위해 파이썬의 실행 경로를 확인한다.
- Mac / Linux:
```bash
which python
```
출력된 경로에 `.venv`가 포함되어 있으면 정상적으로 활성화된 것이다.
- Windows:
```bash
where python
```
출력된 여러 경로 중 최상단에 `.venv` 경로가 나타나면 정상이다.
활성화되지 않은 상태라면 시스템에 설치된 파이썬의 기본 경로가 출력된다.

### 4. 패키지 설치
가상환경이 활성화된 상태에서 패키지를 설치한다.
```bash
pip install requests numpy
```

### 5. 가상환경 비활성화
작업을 마치고 전역 환경으로 돌아가려면 다음 명령어를 입력한다.
```bash
deactivate
```

## 협업과 환경 공유 (requirements.txt)

가상환경 폴더(`.venv`) 자체는 용량이 크고 운영체제 간 호환이 되지 않으므로 GitHub 등에 업로드하지 않는다.
대신 설치된 패키지 목록을 텍스트 파일로 추출하여 공유하는 방식을 채택한다.

### 1. 패키지 목록 추출 (업로드 전)
작업을 마친 후 아래 명령어를 실행하여 설치된 패키지와 버전 정보를 기록한다.
```bash
pip freeze > requirements.txt
```
버전 관리 시스템(Git)에는 무거운 `.venv` 디렉토리를 제외하고(`.gitignore` 처리) 이 `requirements.txt` 파일만 업로드한다.

### 2. 타 환경에서 복원 (다운로드 후)
다른 개발자가 프로젝트를 클론한 후, 본인의 로컬 환경에서 가상환경을 새로 생성하고 활성화한 뒤 아래 명령어를 통해 동일한 패키지 환경을 일괄 구축한다.
```bash
pip install -r requirements.txt
```

## 구현 및 활용 요약
- `.venv`는 프로젝트별로 패키지를 분리하여 관리하는 독립된 샌드박스다.
- 의존성 관리를 위해 선택이 아닌 필수로 활용된다.
- 외부 공유 시에는 폴더 자체를 복사하지 않고 `requirements.txt`를 통해 패키지 의존성을 명시하여 동일한 환경을 재구성하도록 한다.
